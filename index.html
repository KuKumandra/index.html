Панель управления данными

📘 Объяснение работы двух функций Firebase

Функция 1: Добавление данных (Create)

// Код функции добавления: document.getElementById('mainBtn').onclick = async () => { try { await addDoc(collection(db, "requests"), { name: document.getElementById('name').value, email: document.getElementById('email').value, likes: 0, timestamp: serverTimestamp() }); } catch (e) { console.error("Ошибка:", e); } };

Эта функция использует addDoc() для создания нового документа в коллекции "requests". Данные берутся из полей ввода, автоматически добавляется счетчик лайков (0) и временная метка serverTimestamp(), которая устанавливается сервером Firebase для точности времени.

Функция 2: Обновление лайков (Update)

// Код функции обновления: window.processLike = async (id) => { const docRef = doc(db, "requests", id); await updateDoc(docRef, { likes: increment(1) }); };

Эта функция использует updateDoc() с атомарной операцией increment(1). Она принимает ID документа, создает ссылку на него через doc() и увеличивает значение поля "likes" на 1. Важно: increment() гарантирует корректность подсчета даже при одновременных нажатиях нескольких пользователей.

🔍 Дополнительно: Чтение в реальном времени (Read)

// Функция чтения: const q = query(collection(db, "requests"), orderBy("timestamp", "desc")); onSnapshot(q, (snapshot) => { // Автоматически обновляется при изменениях });

Хотя это не требуется по заданию, здесь используется onSnapshot() для автоматического обновления интерфейса при любых изменениях в базе данных. Это демонстрирует мощь Firebase Real-time функциональности.

📊 Структура документа в Firestore:

  • name (string) - имя пользователя
  • email (string) - email пользователя
  • likes (number) - счетчик лайков
  • timestamp (timestamp) - время создания

⚙️ Принцип работы: При нажатии кнопки "Опубликовать" срабатывает первая функция - создается новая запись. При нажатии на звездочку у любой записи срабатывает вторая функция - счетчик лайков увеличивается. Обе функции асинхронные и используют современный синтаксис async/await для обработки промисов.
